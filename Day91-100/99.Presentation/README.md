# Day 99: 项目展示与复盘

本项目 (**DTask**) 的总结、展示与复盘阶段。

## 目标

1.  **项目展示**: 梳理项目架构、功能和亮点，准备演示材料。
2.  **技术复盘**: 总结开发过程中的经验教训，识别改进点。

## 完成内容

### 1. 演示文档 (`PRESENTATION.md`)

-   **项目概览**: 介绍了 DTask 的定位（分布式任务调度系统）。
-   **架构图**: 展示了 Client -> Server (gRPC/HTTP) -> Raft Consensus -> Storage 的数据流向。
-   **核心特性**: 列举了高可用、强一致性、水平扩展等关键技术点。
-   **性能指标**: 展示了压测数据（QPS、延迟）。

### 2. 复盘报告 (`REVIEW.md`)

-   **技术难点**:
    -   OpenRaft 版本的 API 适配问题。
    -   Rust 异步编程中的生命周期和锁竞争问题 (SQLite Locking)。
-   **做得好的**: 项目结构清晰，测试覆盖率逐步提升。
-   **待改进**: 数据库并发写入性能（SQLite WAL 模式优化），分布式节点间的动态发现。
