## 报告漏洞

如果你发现了安全漏洞，请**不要**公开创建 issue。

### 报告流程

1. **私密报告**
   - 发送邮件到：security@example.com
   - 或使用 GitHub 的私密漏洞报告功能

2. **包含信息**
   - 漏洞描述
   - 重现步骤
   - 影响范围
   - 可能的修复方案（如果有）

3. **响应时间**
   - 我们会在 48 小时内确认收到
   - 在 7 天内提供初步评估
   - 在 30 天内发布修复（如果可能）

### 安全更新

安全更新会通过以下方式发布：

- GitHub Security Advisories
- 新的补丁版本
- CHANGELOG 中的安全说明

## 安全最佳实践

### 依赖管理

```bash
# 定期审计依赖
cargo audit

# 更新依赖
cargo update

# 检查过时的依赖
cargo outdated
```

### 代码审查

- 所有代码都需要经过审查
- 使用 Clippy 检查常见问题
- 运行所有测试

### 构建安全

- 使用官方 Rust 镜像
- 验证依赖的完整性
- 最小化 Docker 镜像
